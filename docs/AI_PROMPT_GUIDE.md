## 概述

TrendRadar 的 AI 分析使用 `config/ai_analysis_prompt.txt` 中定义的提示词模板。该提示词将 AI 定位为**高级情报分析师**，专注于从公开信息中提取有价值的洞察。

### 设计理念

```
传统新闻聚合          vs          TrendRadar AI 分析
─────────────────────────────────────────────────────
罗列新闻标题                      提炼共性叙事
情感二元判断                      识别舆论断层
单点信息描述                      跨平台关联分析
"建议关注"废话                   具体行动指南
```

## 核心思维模型

提示词定义了 4 个核心思维模型，指导 AI 如何分析信息：

### 1. 见微知著 (Signal Detection)

```
目标：从冷门信息中发现与热门事件的关联

示例：
  热门：某科技公司股价暴涨 (排名 #1)
  冷门：该公司专利申请数量激增 (排名 #50)
  关联：专利布局预示产品突破，股价上涨有基本面支撑
```

### 2. 交叉验证 (Triangulation)

```
目标：利用不同信息源的差异发现认知套利机会

对比维度：
  热榜 = 大众情绪、流量驱动、易受操纵
  RSS  = 专家视角、深度分析、滞后但准确

示例：
  微博热搜："某币暴涨，财富密码来了！"（大众狂欢）
  专业RSS：  "监管部门约谈主要交易所"  （风险预警）
  结论：舆论狂热与监管收紧形成反差，存在风险
```

### 3. 反直觉思考 (Counter-Intuitive)

```
目标：在共识中寻找机会，在恐慌中保持冷静

原则：
  全网叫好时 → 寻找风险
  全网恐慌时 → 寻找机会
  拒绝平庸共识

示例：
  现象：全网唱衰某行业
  反思：是否已经触底？头部企业是否在逆势布局？
```

### 4. 结构化输出 (MECE)

```
MECE = Mutually Exclusive, Collectively Exhaustive
     = 相互独立，完全穷尽

目标：确保分析维度不重叠、不遗漏

错误示例：
  1. 经济影响
  2. 股市影响  ← 与 #1 重叠
  3. 社会影响

正确示例：
  1. 宏观经济（GDP、通胀）
  2. 资本市场（股市、债市）
  3. 民生消费（就业、物价）
```

## 分析板块详解

AI 输出包含 5 个核心板块，每个板块有明确的分析任务：

### 1. 核心热点态势 (core_trends)

```yaml
任务: 提炼共性叙事，而非简单罗列

结构:
  - 一句话定性（使用：全网霸屏/破圈扩散/圈层热点）
  - 【宏观主线】：挖掘底层逻辑
  - 【微观领域】：细分热点

跨平台分级标准:
  全网霸屏: 5+ 平台同时上榜
  破圈扩散: 3-4 平台同时上榜
  圈层热点: 1-2 平台火爆

示例输出:
  "AI概念股持续破圈扩散，科技与金融形成共振。
   【宏观主线】：大模型落地加速引发的资本重估
   【微观领域】：
   1. 硬件：算力芯片供不应求
   2. 应用：AI Agent 创业潮涌现"
```

### 2. 舆论风向争议 (sentiment_controversy)

```yaml
任务: 绘制情绪光谱，识别舆论断层

避免: 简单的"褒/贬"二元判断

结构:
  - 【情绪光谱】：主流声音 vs 潜流暗涌
  - 【核心矛盾】：利益冲突 or 认知分歧

示例输出:
  "【情绪光谱】：媒体力挺政策利好，但底层评论充斥焦虑情绪
   【核心矛盾】：
   1. 利益冲突：企业降本增效 vs 员工就业稳定
   2. 认知分歧：'狼来了'预警疲劳 vs 真实风险累积"
```

### 3. 异动与弱信号 (signals)

```yaml
任务: 捕捉时间轴和空间轴上的异常

维度:
  - 【跨平台共振/温差】：多平台联动或差异
  - 【轨迹突变】：急升/衰退/回榜模式
  - 【弱信号捕捉】：早期征兆

轨迹模式识别:
  急升/爆发: 排名快速上升（如 20→3）
  衰退/僵尸: 排名持续下降无反弹（如 10→15→20）
  回榜/反转: 脱榜后复活（如 5→0→2）

示例输出:
  "【跨平台温差】某AI产品在技术社区(知乎)热议，但大众平台(微博)反应冷淡，
   说明当前仍处于'专业圈层讨论'阶段，尚未破圈
   【轨迹突变】某政策话题在15:00呈现急升态势，从#35直冲#3，
   可能有重大进展或官方回应"
```

### 4. RSS 深度洞察 (rss_insights)

```yaml
任务: 寻找"热榜盲区"，提供专业视角补充

策略:
  - 去重：忽略与热榜重复的内容
  - 互补：挖掘技术参数、深度行研
  - 前瞻：识别早期行业信号

结构:
  - 【认知纠偏】：专业视角如何修正大众误区
  - 【硬核增量】：热榜缺失的关键数据

示例输出:
  "【认知纠偏】热搜炒作'某技术重大突破'，但专业论文显示该进展
   是渐进式改良而非颠覆性创新，市场预期可能过热
   【硬核增量】行业报告披露核心指标：用户留存率仅17%，
   远低于市场预期的40%，存在商业化瓶颈"
```

### 5. 研判策略建议 (outlook_strategy)

```yaml
任务: 基于推演给出具体行动指南

禁止: "建议持续关注"等废话

结构: 针对不同受众的具体建议
  - 投资者：风口捕捉 or 风险预警
  - 品牌方：流量借势 or 公关避坑
  - 公众：认知纠偏 or 生活决策

示例输出:
  "1. 投资者：AI算力板块短期过热，建议等待回调后分批建仓，
     关注算力租赁而非纯硬件标的
   2. 品牌方：'AI+'概念已成流量密码，但需避免空喊口号，
     应展示具体落地场景
   3. 公众：警惕'AI替代焦虑'贩卖，当前更多是辅助工具而非替代品"
```

## 输出格式说明

AI 输出为 JSON 格式，便于程序解析和多渠道分发：

```json
{
  "core_trends": "核心热点态势分析（200字内）",
  "sentiment_controversy": "舆论风向争议分析（100字内）",
  "signals": "异动与弱信号（150字内）",
  "rss_insights": "RSS深度洞察（100字内）",
  "outlook_strategy": "研判策略建议"
}
```

### 格式规范

1. **禁止 Markdown**：输出纯文本，不使用 `**加粗**` 等格式
2. **使用【】分段**：用中文方括号标记段落标题
3. **序号规则**：使用 `1. 2. 3.` 序号时，行内不能再用 `【】`
4. **字数控制**：每个板块有字数限制，确保简洁

## 可用变量

在分析时，以下变量会被实际数据替换：

| 变量 | 说明 | 示例 |
|------|------|------|
| `{language}` | 输出语言 | `中文` |
| `{report_mode}` | 报告模式 | `incremental` |
| `{report_type}` | 报告类型描述 | `增量监控模式` |
| `{current_time}` | 当前时间 | `2026-02-02 10:30:00` |
| `{news_count}` | 热榜新闻条数 | `15` |
| `{rss_count}` | RSS 新闻条数 | `8` |
| `{keywords}` | 匹配的关键词 | `石油, 伊朗, 美国` |
| `{platforms}` | 数据来源平台 | `同花顺7x24, 微博` |
| `{news_content}` | 热榜新闻内容 | （新闻列表） |
| `{rss_content}` | RSS 订阅内容 | （RSS 文章列表） |

## 自定义指南

### 修改分析风格

如果希望调整 AI 的分析风格，可以修改 `[system]` 部分：

```
# 更激进的投资导向
将"高级情报分析师"改为"对冲基金研究员"
强调：alpha机会识别、做空信号、套利策略

# 更保守的风险导向
将"高级情报分析师"改为"风控合规官"
强调：风险预警、监管动态、负面信号
```

### 调整输出板块

如果只需要部分分析板块，修改 `[user]` 部分的 JSON 模板：

```json
// 精简版：只保留 3 个板块
{
  "core_trends": "...",
  "signals": "...",
  "outlook_strategy": "..."
}
```

### 添加自定义维度

在 JSON 输出中添加新字段：

```json
{
  "core_trends": "...",
  "crypto_analysis": "加密货币专项分析（如需要）",
  "outlook_strategy": "..."
}
```
